# AI Usage

## Overview

This repository is part of a broader AI-assisted development workflow documented in my public playbook: [Android AI Workflow Playbook](https://github.com/JaviGarcia1995/android-ai-workflow-playbook).

That playbook explains the general methodology, workflow structure, and validation rules used when working with AI tools in Android projects.

## How AI Was Used

In this project, AI was used as a supporting engineering assistant for:

- drafting and refining documentation
- structuring technical explanations
- improving public-facing wording for the case study
- helping organize architecture and decision summaries

## What AI Was Not Used For

AI was not used to replace engineering judgment or ownership.

It was not used to:
- define the product direction
- make final architecture decisions
- bypass manual validation
- replace code review or technical understanding

## Human Review and Validation

All project-specific content was reviewed and adapted manually to match the actual implementation.

The final documentation reflects the real structure, decisions, and constraints of the project rather than AI-generated assumptions.

## Why This Matters

The goal of this repository is transparency.

This document clarifies where AI helped, where it did not, and how human judgment remained in control.
